### 4) 视觉设计要求（非常重要）

- 主色调：浪漫粉（多层渐变粉、奶油粉、玫瑰粉），搭配白色与一点金色点缀
- 设计风格：**“一眼很浪漫”**，柔光、轻纱、发光粒子、玫瑰元素丰富
- 页面包含很多玫瑰：可用
  - CSS 玫瑰图形（简化版）
  - SVG 玫瑰（推荐）
  - 或者轻量 PNG/WebP（但要小）
- **最终要有动态运动效果**：
  1. 顶部/背景：漂浮玫瑰花瓣（缓慢飘落、左右摆动、轻微旋转）
  2. 页面底部：进入尾声时出现“玫瑰雨/玫瑰环绕”效果（更密集、更浪漫）
  3. 鼠标/触摸交互：点击出现一朵小玫瑰或爱心波纹（轻量）
- 动画实现优先：
  - CSS keyframes（低成本）
  - Canvas（只用一个 canvas，粒子数量可控，移动端降级）
- 必须考虑性能：移动端限制粒子数量，`prefers-reduced-motion` 时降低动画

### 7) 动态计算规则（必须准确、时区要处理）

- 以 **用户本地时区** 展示更符合真实感（前端用 `new Date()`）
- Worker 的 `/api/stats` 也返回同样数据（用于校验/SSR 不要求）
- 对日期建议按本地零点处理，避免时区偏差：
  - 例如把 `2020-08-21` 作为本地 `00:00:00` 计算差值
- 输出字段建议：

```
{
  "met": {"days": 0, "hours": 0, "minutes": 0, "seconds": 0, "totalSeconds": 0},
  "married": {"days": 0, "totalDays": 0},
  "baby": {"days": 0, "totalDays": 0},
  "dates": {"met": "2020-08-21", "married": "2021-01-18", "baby": "2024-12-30"}
}
```
